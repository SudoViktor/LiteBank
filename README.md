# LiteBank
Високонавантажена подієво-орієнтована банківська платформа, побудована на мікросервісній архітектурі. Система використовує Apache Kafka для асинхронної обробки транзакцій, що забезпечує високу продуктивність та надійну узгодженість даних між сервісами.

# Архітектура та інфраструктура
Проєкт складається з набору незалежних мікросервісів, кожен з яких ізольований у власному Docker-контейнері. Взаємодія між сервісами реалізована за допомогою REST API (FastAPI) та брокера повідомлень Kafka.

Бази даних (PostgreSQL)
Система використовує чотири окремі ізольовані бази даних для забезпечення логічного розділення відповідальності:
  auth_service_db — зберігає дані користувачів та облікові записи.
  accounts_db — відповідає за керування банківськими рахунками та випущеними картками.
  transactions_db — забезпечує створення транзакцій та доступ до історії їх статусів.
  ledger_db — «джерело істини» (Source of Truth) для фінансових показників. Тут зберігається повна історія операцій, на основі якої формується та верифікується поточний баланс кожного рахунку.

Брокер повідомлень (Apache Kafka)
Kafka забезпечує асинхронну взаємодію між сервісами за допомогою наступних топіків:
  AccountOwners — сервіс accountservice повідомляє про створення нового рахунку. Це дозволяє transactionservice верифікувати право власності користувача на IBAN перед ініціацією переказу.
  CreateTransaction — сервіс transactionservice публікує запит на створення транзакції для подальшої обробки сервісом ledgerservice.
  CompleteTransaction — сервіс ledgerservice повідомляє transactionservice про фінальний результат обробки (успіх або відмова), щоб той міг оновити статус операції.


Мікросервіси
  authservice — керує реєстрацією, автентифікацією та авторизацією користувачів (JWT).
  accountservice — забезпечує життєвий цикл рахунків та банківських карток.
  transactionservice — API-шлюз для створення транзакцій та відстеження їх стану.
  ledgerservice — ядро системи, що відповідає за фінальну обробку транзакцій та розрахунок актуального балансу на основі реєстру операцій.
  front — веб-інтерфейс (клієнтська частина) для взаємодії користувача з банківською системою.
