# 🏦 LiteBank: Подієво-орієнтована банківська платформа

[![Architecture](https://img.shields.io/badge/Architecture-Microservices-blue.svg)]()
[![Tech](https://img.shields.io/badge/Tech-FastAPI%20%7C%20Kafka%20%7C%20PostgreSQL-green.svg)]()

**LiteBank** — це високонавантажена банківська система, побудована на сучасній мікросервісній архітектурі. Платформа спеціалізується на асинхронній обробці фінансових транзакцій із використанням Apache Kafka, що гарантує високу продуктивність та бездоганну цілісність даних.

---

## 🏗 Архітектура та інфраструктура
Проєкт складається з набору незалежних мікросервісів, кожен з яких ізольований у власному **Docker**-контейнері. Взаємодія між компонентами реалізована через **REST API (FastAPI)** та подієво-орієнтований підхід за допомогою **Apache Kafka**.

### 🗄 Бази даних (PostgreSQL)
Для забезпечення логічного розділення відповідальності та масштабованості кожна частина системи має власну базу даних:

* 🔒 **`auth_service_db`**: Зберігає дані користувачів та облікові записи.
* 💳 **`accounts_db`**: Відповідає за життєвий цикл рахунків та емісію банківських карток.
* 📜 **`transactions_db`**: Забезпечує створення транзакцій та відстеження їх статусів.
* ⚖️ **`ledger_db`**: «Джерело істини» (Source of Truth). Тут зберігається повна історія всіх успішних операцій, на основі якої формується та верифікується актуальний баланс рахунків.

---

## 📨 Взаємодія через Kafka
Брокер повідомлень Kafka забезпечує надійний асинхронний зв'язок між сервісами через такі топіки:

* **`AccountOwners`**: Сповіщення про створення нового рахунку. Дозволяє `transactionservice` верифікувати право власності користувача на IBAN перед ініціацією переказу.
* **`CreateTransaction`**: Запит на створення та обробку транзакції від `transactionservice` до `ledgerservice`.
* **`CompleteTransaction`**: Сповіщення про результат обробки (успіх/відмова), що дозволяє `transactionservice` оновити фінальний статус операції.

---

## 🧩 Мікросервіси

* 🔐 **`authservice`**: Автентифікація та авторизація користувачів за допомогою JWT.
* 🏦 **`accountservice`**: Управління рахунками та банківськими картками.
* 💸 **`transactionservice`**: API-шлюз для користувачів — створення транзакцій та моніторинг їх стану.
* ⚙️ **`ledgerservice`**: Ядро системи, відповідальне за фінальну обробку фінансових операцій та розрахунок балансу.
* 🖥 **`front`**: Веб-інтерфейс (SPA), що надає користувачу повний контроль над його фінансами.
